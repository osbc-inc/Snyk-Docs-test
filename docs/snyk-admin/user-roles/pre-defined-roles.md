# 사전 정의된 역할

Snyk은 Snyk 웹 UI 또는 [Snyk API](../user-management-with-the-api/)를 사용하여 할당하고 관리할 수 있는 표준 사용자 역할 세트를 제공합니다. 사전 정의된 역할의 권한 세트는 사용자 정의할 수 없습니다. 대신, Snyk은 Snyk 웹 UI의 '관리 역할'에서 사용자 정의 역할을 생성하는 것을 권장합니다.

Snyk이 제공하는 사전 정의된 역할은 다음과 같습니다:

- **조직 관리자:** 팀 리더에 해당하는 표준 역할입니다. 이 역할을 가진 사용자는 프로젝트를 추가 및 삭제할 수 있으며, Snyk 검사를 무시하거나 그룹 구성원에게 조직 수준 역할을 제공할 수 있습니다.
- **조직 협력자:** 개발자에 해당하는 표준 역할입니다. 이 역할은 소규모 팀이나 개발자 중심 조직 접근 방식에 이상적입니다.
- **그룹 관리자:** 회사 내의 Snyk 사용을 전반적으로 감독하는 사람을 위한 표준 역할로, 그룹 및 조직 수준에서 전체 권한 세트를 제공합니다. 이는 그룹 관리자가 그룹 아래에 있는 모든 조직에 대해 자동으로 조직 관리자임을 의미하지만, 조직 수준 목록에서는 표시되지 않습니다.
- **그룹 뷰어:** 그룹 수준에 액세스할 수 있지만 Snyk에서 작업을 수행하려면 조직 수준 권한이 필요한 사용자입니다. 이는 주로 Snyk과의 온보딩 중에 그룹 권한에 연결된 기능을 이해하고 배포 후 사용을 위해 사용자 정의 그룹 역할을 설계하는 초기 단계로 사용됩니다.
- **그룹 구성원:** 기본적으로 작동하지 않는 그룹 수준 역할입니다. 구성원은 그룹에 액세스하지만, Snyk을 효과적으로 사용하려면 조직 수준 권한이 필요합니다. 이 역할은 그룹의 액세스를 제한하고 특정 조직에 액세스를 부여하는 데 이상적입니다.

{% hint style="info" %}
**기능 가용성**

그룹 수준 역할은 엔터프라이즈 엔터프라이즈 플랜에서만 사용할 수 있습니다. 자세한 정보는 [요금제 및 가격](https://snyk.io/plans/)을 참조하십시오.
{% endhint %}

## 역할 유형

역할은 조직 수준과 그룹 수준에서 모두 관리할 수 있습니다.

그룹 수준 역할은 조직 및 그룹 수준에서 권한을 포함합니다. 그룹 역할에 추가된 모든 조직 권한은 그룹에 있는 모든 조직에 부여됩니다. 예를 들어, 사전 정의된 그룹 역할 **그룹 뷰어**는 사용자에게 그룹에서의 선택적인 보기 권한과 또한 해당 역할이 있으면 그룹에 속하는 모든 조직에 대해 읽기 전용 액세스 권한을 부여합니다.

조직 역할은 오직 조직 수준에서만 권한을 가지고 있습니다. 조직 역할은 특정 조직에 대한 특정 권한을 부여하는 데 유용합니다.

특정 역할에 대해 그룹 및 조직 수준 역할의 결합을 사용하여 모든 역할에서 특정 액세스를 부여할 수 있습니다. 예를 들어, 특정 조직에 대해 사용자에게 **조직 관리자** 액세스를 부여하고, 다른 그룹에 있는 모든 조직에 대해 읽기 전용 액세스를 부여하려면 사용자에게 그룹 수준에서 **그룹 뷰어** 역할 및 선택한 조직에 대해 **조직 관리자** 역할을 부여할 수 있습니다.
